# SSH Router Reboot System สำหรับ GL.iNet GL-XE300C4

ระบบนี้จะทำการรีบูตเร้าเตอร์ GL.iNet GL-XE300C4 อัตโนมัติทุกๆ 10 ไอดี (หรือตามที่ตั้งค่า) เพื่อเปลี่ยน IP Address ใหม่

## ✨ ฟีเจอร์

- 🔄 รีบูตเร้าเตอร์อัตโนมัติทุกๆ N ไอดีที่สำเร็จ (ตั้งค่าได้)
- 🔑 ระบบ SSH Key authentication แบบไม่ต้องใส่รหัสผ่าน
- 🌐 ตรวจสอบ IP Address ใหม่หลังรีบูต
- ⚙️ ตั้งค่าได้ผ่านไฟล์ config
- 📊 แสดงสถานะการทำงานแบบ Real-time

## 🔧 การติดตั้ง

### 1. ติดตั้ง Dependencies
```bash
python install_requirements.py
```

### 2. เปิดใช้งาน SSH ในเร้าเตอร์
1. เข้าไปที่หน้า Admin Panel ของเร้าเตอร์ (โดยปกติ http://192.168.8.1)
2. ไปที่ System > Administration
3. เปิดใช้งาน SSH Access
4. ตั้งรหัสผ่าน root (หรือเว้นว่างไว้หากใช้ default)

### 3. แก้ไขไฟล์ config_register.txt
เพิ่มบรรทัดเหล่านี้ในไฟล์ config:
```
# Router SSH Configuration for GL.iNet GL-XE300C4
ROUTER_SSH_HOST=192.168.8.1
ROUTER_SSH_PORT=22
ROUTER_SSH_USERNAME=root
ROUTER_REBOOT_ENABLED=True
ROUTER_REBOOT_INTERVAL=10
```

## 🚀 การใช้งาน

### วิธีที่ 1: Setup อัตโนมัติ
1. รันโปรแกรมหลัก
2. เลือก "s" จากเมนูเพื่อ Setup SSH
3. ใส่รหัสผ่านเร้าเตอร์เมื่อถูกถาม
4. ระบบจะ setup SSH key โดยอัตโนมัติ

### วิธีที่ 2: Setup ด้วยมือ
```bash
# สร้าง SSH key
ssh-keygen -t rsa -b 2048 -f ~/.ssh/id_rsa -N ""

# คัดลอก public key ไปเร้าเตอร์
ssh-copy-id root@192.168.8.1
```

### ทดสอบการเชื่อมต่อ
จากเมนูหลัก เลือก "t" เพื่อทดสอบการเชื่อมต่อ SSH

## ⚙️ การตั้งค่า

| ตัวแปร | ค่าเริ่มต้น | คำอธิบาย |
|--------|------------|----------|
| ROUTER_SSH_HOST | 192.168.8.1 | IP Address ของเร้าเตอร์ |
| ROUTER_SSH_PORT | 22 | Port SSH |
| ROUTER_SSH_USERNAME | root | Username สำหรับ SSH |
| ROUTER_REBOOT_ENABLED | True | เปิด/ปิดการรีบูตอัตโนมัติ |
| ROUTER_REBOOT_INTERVAL | 10 | จำนวนไอดีก่อนรีบูต |

## 📊 การทำงาน

1. **การนับไอดี**: ระบบจะนับเฉพาะไอดีที่สมัครสำเร็จเท่านั้น
2. **การรีบูต**: เมื่อครบจำนวนที่ตั้งไว้ ระบบจะ:
   - ส่งคำสั่ง reboot ผ่าน SSH
   - รอ 60 วินาทีให้เร้าเตอร์รีบูต
   - ตรวจสอบว่าเร้าเตอร์กลับมาออนไลน์แล้ว
   - ตรวจสอบ IP Address ใหม่
3. **Reset Counter**: หลังรีบูตสำเร็จ ตัวนับจะกลับเป็น 0

## 🛠️ การแก้ไขปัญหา

### ปัญหา SSH Connection Failed
```bash
# ตรวจสอบการเชื่อมต่อ
ping 192.168.8.1

# ตรวจสอบ SSH service
ssh root@192.168.8.1
```

### ปัญหา Permission Denied
```bash
# ตรวจสอบสิทธิ์ SSH key
chmod 600 ~/.ssh/id_rsa
chmod 644 ~/.ssh/id_rsa.pub
```

### ปัญหาเร้าเตอร์ไม่กลับมาออนไลน์
- ตรวจสอบสายแลน
- รอเพิ่มเติม (เร้าเตอร์บางรุ่นใช้เวลานาน)
- รีสตาร์ทด้วยมือ
